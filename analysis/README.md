# Analysis (Pandas First)

This folder contains the first-pass analysis workflow for DiceLab benchmark data using pandas.

## Why pandas first?

- Familiar, industry-standard dataframe tooling
- Excellent for reproducible tabular transformations
- Easy handoff into notebooks, scripts, and Streamlit later

## Inputs

- `benchmarks/results/benchmark_report.json`

## Module Structure

| File | Role |
|---|---|
| `build_tables.py` | Core transformation library — loads the benchmark JSON and normalises it into the three canonical DataFrames (`run_table`, `batch_table`, `metadata_table`). Importable from scripts, notebooks, and a future Streamlit dashboard. |
| `run_analysis.py` | CLI driver — calls `build_tables`, exports CSVs to `shared-data/`, and writes the markdown summary to `reports/`. |
| `analyze_tables.py` | Analysis and visualization library — loads the canonical CSVs and exposes Plotly chart functions for within-language scaling, trial distributions, cross-language comparisons, consistency (CV), and macro batch trends. Returns `plotly.graph_objects.Figure` objects compatible with both HTML export and Streamlit. |
| `build_report.py` | HTML report builder — assembles all Plotly figures into a self-contained, single-file HTML report with embedded CSS/JS dropdown navigation. Writes to `reports/benchmark_report.html`. |

## Outputs

Generated by `analysis/run_analysis.py`:

- `shared-data/analysis_run_table.csv`
- `shared-data/analysis_batch_table.csv`
- `shared-data/analysis_language_workload_summary.csv`
- `reports/analysis_summary.md`

Generated by `analysis/build_report.py`:

- `reports/benchmark_report.html`

## Run

From repo root:

```bash
python analysis/run_analysis.py
```

Optional:

```bash
python analysis/run_analysis.py --input benchmarks/results/benchmark_report.json
```

To build the HTML report (requires CSVs in `shared-data/` from the step above):

```bash
python analysis/build_report.py
```

Optional:

```bash
python analysis/build_report.py --shared-data shared-data --output reports/benchmark_report.html
```

## Notes

- This is a scripted data preparation baseline, not a dashboard.
- The data model is intentionally reusable for future Streamlit/report layers.
